Angular Routing

Angular is designed to build single-page applications (SPAs) while also handling routing by showing or hiding portions of the display that correspond to particular components.

To handle routing in Angular, you can use the Router along with a few other features, but for simple routing, the Router is sufficient.

Basic Routing

Basic routing can be handled with simple routes.
First in app-routing.module.ts define your routes like so 

let routes: Routes = [
    { path: 'home', component: HomeComponent },
    { path: ' ', redirectTo: '/home', pathMatch: 'full'},
];

Basic routes pattern is { path: "URL path", component: "Component that should be displayed" }

Then you can choose one of few attempts to navigate to another URL.

  1. The simplest one is just adding 
    <a [routerLink]="['/home']">Home</a>
  2. Another way of routing is via button click (this is approach that I took creating this site) 
    <button (click)="navigateToHome()">
    Then in your component function navigateToHome() need to be implemented. First Router is needed, so inject it in constructor, then implement method with help of router. 
    constructor(private router: Router) {}

navigateToHome() {
    this.router.navigate(['/home']);
}

Dynamic Routing

Dynamic Routing is very useful when it comes to routing to specific id/tag/user/item. You can define a base route and then display context based of on url.

First define route in app-routing.module.ts 
{ path: 'frontend/routing/details/:id', component: RoutingComponent}
The :id determinate which part of route should be replaced.
In your component.ts Add Router in constructor add implement an event when navigation should be executed. In example below I implemented on button click. 
constructor(private router: Router){
    }
  
protected navigate(id: number) {
    this.router.navigate(['frontend/routing/details/', id])
}

I used simple and basic routing but the idea is similar when you want to implement more complicated routes. Here I specifically type whole route because of the fact that this app handles routing in different way which I explain later

Child Routes

Next step in routing is adding child routes. Child routing helps with organize paths and gives them hierarchy so user can see the actual path of view.
First of all define in app-routing.module.ts your child routes. 

{ path: 'frontend/routing', component: RoutingComponent, children:[
    {path: 'firstChild', component: RoutingComponent},
    {path: 'secondChild', component: RoutingComponent}
]}

Next in component that's gonna handle routing, import ActivatedRoute from @angular/router and inject it in constructor. Then create functions to handle routing navigation

constructor(private router: Router, private route: ActivatedRoute){         
}

protected navigateToFirstChild() {
    this.router.navigate(['firstChild'], {relativeTo: this.route});
}

protected navigateToSecondChild() {
    this.router.navigate(['secondChild'], {relativeTo: this.route});
}
